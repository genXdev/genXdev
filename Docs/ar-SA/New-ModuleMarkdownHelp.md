# New-ModuleMarkdownHelp

> **Module:** GenXdev.Coding.PowerShell.Modules | **Type:** Function | **Aliases:** —

## Synopsis

> ينشئ ملفات تعليمات Markdown غنية لأي وحدة PowerShell.

## Description

ينشئ موقع مساعدة شامل بتنسيق Markdown لأي وحدة PowerShell النمطية:
ملف .md واحد لكل أمر cmdlet مع جميع أقسام البيانات الوصفية، بالإضافة إلى ملف README.md 
فهرس مع جداول لكل وحدة فرعية ترتبط بكل ملف أمر cmdlet.

اكتشاف الوحدة الفرعية محمول: أدلة dot-source .psm1 للأوامر النصية cmdlets، 
مساحات أسماء .NET للأوامر المترجمة cmdlets، مع سلسلة احتياطية متعددة المستويات 
للوحدات النمطية التي لا تحتوي على هيكل وحدة فرعية.

## Syntax

```powershell
New-ModuleMarkdownHelp -ModuleName <String> [-ApiEndpoint <String>] [-ApiKey <String>] [-Force] [-Language <String>] [-LinkPrefix <String>] [-Model <String>] [-NoSupportForJsonSchema <String>] [-OutputPath <String>] [-SkipTranslation] [-TranslationInstructions <String>] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-ModuleName` | String | ✅ | — | 0 | — | اسم وحدة PowerShell النمطية التي سيتم إنشاء تعليمات المساعدة لها |
| `-OutputPath` | String | — | — | Named | — | دليل إخراج مخصص لملفات .md. الافتراضي هو <moduleRoot>\Docs\<language>. |
| `-Language` | String | — | — | Named | `'en-US'` | ar-SA |
| `-Force` | SwitchParameter | — | — | Named | — | استبدال ملفات .md الموجودة دون طلب تأكيد |
| `-SkipTranslation` | SwitchParameter | — | — | Named | — | @{response=Skip LLM translation; keep help in source language} |
| `-Model` | String | — | — | Named | — | معرف النموذج أو النمط المستخدم للترجمات بالذكاء الاصطناعي |
| `-ApiEndpoint` | String | — | — | Named | — | رابط نقطة نهاية API للترجمات بالذكاء الاصطناعي |
| `-ApiKey` | String | — | — | Named | — | مفتاح API للترجمات المعتمدة بالذكاء الاصطناعي |
| `-NoSupportForJsonSchema` | String | — | — | Named | — | يشير إلى أن LLM لا يدعم مخططات JSON |
| `-LinkPrefix` | String | — | — | Named | — | بادئة عنوان URL لروابط فهرس README (على سبيل المثال، https://github.com/org/repo/Docs/) |
| `-TranslationInstructions` | String | — | — | Named | — | وظيفة `Set-ADUser` هي cmdlet في PowerShell يُستخدم لتعديل خصائص مستخدم Active Directory. |

## Examples

### New-ModuleMarkdownHelp -ModuleName 'Pester' -SkipTranslation

```powershell
New-ModuleMarkdownHelp -ModuleName 'Pester' -SkipTranslation
```

يُنشئ مجلد Docs\ مع ملف .md واحد لكل أمر cmdlet من Pester بالإضافة إلى فهرس README.md.

### New-ModuleMarkdownHelp -ModuleName 'GenXdev' -Language 'nl-NL' -Force -Model 'deepseek-v4-pro' -ApiKey 'your-api-key' -ApiEndpoint 'https://api.deepseek.com/chat/completions'

```powershell
New-ModuleMarkdownHelp -ModuleName 'GenXdev' -Language 'nl-NL' -Force -Model 'deepseek-v4-pro' -ApiKey 'your-api-key' -ApiEndpoint 'https://api.deepseek.com/chat/completions'
```

ينشئ تعليمات مساعدة بتنسيق ماركاون مترجمة للهولندية، مع استبدال الملفات الموجودة.

### New-ModuleMarkdownHelp -ModuleName 'GenXdev' -LinkPrefix `     'https://github.com/genXdev/genXdev/Docs/' -SkipTranslation

```powershell
New-ModuleMarkdownHelp -ModuleName 'GenXdev' -LinkPrefix `
    'https://github.com/genXdev/genXdev/Docs/' -SkipTranslation
```

يقوم بإنشاء تعليمات تحتوي على روابط GitHub مطلقة في فهرس ملف README.
