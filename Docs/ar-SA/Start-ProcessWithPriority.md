# Start-ProcessWithPriority

> **Module:** GenXdev.Windows | **Type:** Cmdlet | **Aliases:** `nice`

## Synopsis

> يبدأ عملية بمستوى أولوية محدد.

## Description

* يُطلق برنامجًا تنفيذيًا بمستوى أولوية قابل للتخصيص ويوفر خيارات للانتظار ومعالجة العمليات.
* يغلف Start-Process بوظائف إضافية للتحكم في أولوية العملية وسلوك التنفيذ.

## Syntax

```powershell
Start-ProcessWithPriority -FilePath <String> [[-ArgumentList] <String[]>] [[-Priority] <String>] [-NoWait] [-PassThru] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-FilePath` | String | ✅ | — | 0 | — | المسار إلى الملف القابل للتنفيذ لتشغيله |
| `-ArgumentList` | String[] | — | — | 1 | — | الوسائط التي سيتم تمريرها إلى الملف القابل للتنفيذ |
| `-Priority` | String | — | — | 2 | — | مستوى أولوية العملية |
| `-NoWait` | SwitchParameter | — | — | Named | `False` | لا تنتظر اكتمال العملية |
| `-PassThru` | SwitchParameter | — | — | Named | `False` | أعد كائن العملية |

## Examples

### Example 1

```powershell
Start-ProcessWithPriority -FilePath "notepad.exe" -Priority "Low" -NoWait
```

يبدأ تشغيل تطبيق "المفكرة" بأولوية منخفضة ويعود فورًا.

### Example 2

```powershell
nice notepad.exe -Priority High
```

يستخدم الاسم المستعار 'nice' لبدء عملية بأولوية عالية.
