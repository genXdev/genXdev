# Test-PathUsingWindowsDefender

> **Module:** GenXdev.Windows | **Type:** Cmdlet | **Aliases:** `virusscan`, `HasNoVirus`

## Synopsis

> يقوم بفحص الملفات أو المجلدات بحثًا عن البرامج الضارة باستخدام Windows Defender.

## Description

* يقوم بإجراء فحص مستهدف للملفات أو الدلائل المحددة باستخدام واجهة سطر الأوامر في Windows Defender (MpCmdRun.exe).
* يمكنه المسح في وضع الكشف فقط أو مع تمكين الإصلاح التلقائي للتهديدات.
* يُرجع صحيح إذا لم يتم اكتشاف أي تهديدات، وخاطئ إذا تم العثور على تهديدات أو فشل الفحص.

## Syntax

```powershell
Test-PathUsingWindowsDefender -FilePath <String> [-EnableRemediation] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-FilePath` | String | ✅ | ✅ (ByValue, ByPropertyName) | 0 | — | المسار إلى الملف أو الدليل المراد فحصه |
| `-EnableRemediation` | SwitchParameter | — | — | Named | `False` | يأمر Windows Defender باتخاذ إجراء ضد التهديدات |

## Examples

### Example 1

```powershell
Test-PathUsingWindowsDefender -FilePath "C:\Downloads\file.exe" -Verbose
```

يقوم بفحص الملف المحدد ويعرض إخراجًا مفصلاً.

### Example 2

```powershell
virusscan "C:\Downloads\file.exe" -EnableRemediation
```

يقوم بفحص الملف باستخدام الاسم المستعار وتمكين المعالجة.

### Example 3

```powershell
"C:\Downloads\file.exe" | HasNoVirus
```

يقوم بتوجيه مسار الملف إلى الأمر cmdlet باستخدام اسم مستعار.
