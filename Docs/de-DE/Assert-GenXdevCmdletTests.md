# Assert-GenXdevCmdletTests

> **Module:** GenXdev.Coding.PowerShell.Modules | **Type:** Function | **Aliases:** `improvecmdlettests`

## Synopsis

> Bestätigt und verbessert die Unit-Tests eines angegebenen GenXdev-Cmdlets.

## Description

Diese Funktion hilft, Komponententests für GenXdev-Cmdlets zu warten und zu verbessern, indem:
1. Testdateien erstellt werden, falls sie nicht existieren
2. Das Cmdlet in VS Code geöffnet wird
3. KI-Eingabeaufforderungen für die Generierung/Verbesserung von Tests vorbereitet und angewendet werden
4. Der Workflow zur Testausführung verwaltet wird

## Syntax

```powershell
Assert-GenXdevCmdletTests -CmdletName <String> [[-Prompt] <String>] [[-PromptKey] <String>] [-AssertFailedTest] [-ContinuationHandled] [-EditPrompt] [-FromScripts] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-CmdletName` | String | ✅ | ✅ (ByValue, ByPropertyName) | 0 | — | Der Name des Cmdlets zur Verbesserung von Komponententests |
| `-Prompt` | String | — | — | 1 | `''` | Benutzerdefinierter KI-Prompt-Text zur Verwendung |
| `-PromptKey` | String | — | — | 2 | — | Der KI-Prompt-Schlüssel, der für die Vorlagenauswahl verwendet werden soll |
| `-EditPrompt` | SwitchParameter | — | — | Named | — | Wechseln Sie, um nur die KI-Eingabeaufforderung zu bearbeiten |
| `-AssertFailedTest` | SwitchParameter | — | — | Named | — | Gibt an, einen fehlgeschlagenen Test zu bestätigen |
| `-FromScripts` | SwitchParameter | — | — | Named | — | In Skriptdateien statt Modulen suchen |
| `-ContinuationHandled` | SwitchParameter | — | — | Named | — | The `-ContinuationHandled` parameter. |
