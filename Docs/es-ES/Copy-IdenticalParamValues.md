# Copy-IdenticalParamValues

> **Module:** GenXdev.Helpers | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> Copia los valores de los parámetros enlazados a una nueva tabla hash basada en los parámetros posibles de otra función.

## Description

* Este cmdlet crea una nueva tabla hash que contiene solo los valores de los parámetros
  que coinciden con los parámetros definidos en la función de destino especificada.
* La tabla hash resultante se puede usar para invocar la función mediante
  el splatting de PowerShell.
* Los parámetros de conmutador solo se incluyen en el resultado si se proporcionaron explícitamente
  y se establecieron en $true en los parámetros enlazados.
* Los parámetros de conmutador no presentes se excluyen del resultado para mantener
  la semántica adecuada de los parámetros.

## Syntax

```powershell
Copy-IdenticalParamValues -BoundParameters <Object[]> -FunctionName <String> [[-DefaultValues] <Object>] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-BoundParameters` | Object[] | ✅ | — | 0 | — | Parámetros de origen para copiar |
| `-FunctionName` | String | ✅ | — | 1 | — | Nombre de la función de destino para filtrar parámetros |
| `-DefaultValues` | Object | — | — | 2 | — | Valores predeterminados para parámetros |

## Examples

### Example 1

```powershell
function Test-Function {
    [CmdletBinding()]
    param(
        [Parameter(Mandatory = $true)]
        [string] $Path,
        [Parameter(Mandatory = $false)]
        [switch] $Recurse
    )

    $params = GenXdev\Copy-IdenticalParamValues `
        -BoundParameters $PSBoundParameters `
        -FunctionName 'Get-ChildItem'

    Get-ChildItem @params
}
```

Muestra cómo copiar parámetros enlazados para usarlos con splatting.
