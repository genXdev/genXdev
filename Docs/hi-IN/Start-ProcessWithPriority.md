# Start-ProcessWithPriority

> **Module:** GenXdev.Windows | **Type:** Cmdlet | **Aliases:** `nice`

## Synopsis

> निर्दिष्ट प्राथमिकता स्तर के साथ एक प्रक्रिया प्रारंभ करता है।

## Description

* अनुकूलन योग्य प्राथमिकता स्तर के साथ एक निष्पादन योग्य चलाता है और प्रतीक्षा और प्रक्रिया हैंडलिंग के विकल्प प्रदान करता है।
* प्रक्रिया प्राथमिकता और निष्पादन व्यवहार को नियंत्रित करने के लिए अतिरिक्त कार्यक्षमता के साथ Start-Process को रैप करता है।

## Syntax

```powershell
Start-ProcessWithPriority -FilePath <String> [[-ArgumentList] <String[]>] [[-Priority] <String>] [-NoWait] [-PassThru] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-FilePath` | String | ✅ | — | 0 | — | चलाने के लिए निष्पादन योग्य का पथ |
| `-ArgumentList` | String[] | — | — | 1 | — | निष्पादन योग्य को पास करने के लिए तर्क |
| `-Priority` | String | — | — | 2 | — | प्रक्रिया प्राथमिकता स्तर |
| `-NoWait` | SwitchParameter | — | — | Named | `False` | प्रक्रिया पूर्ण होने की प्रतीक्षा न करें |
| `-PassThru` | SwitchParameter | — | — | Named | `False` | प्रक्रिया ऑब्जेक्ट लौटाएं |

## Examples

### Example 1

```powershell
Start-ProcessWithPriority -FilePath "notepad.exe" -Priority "Low" -NoWait
```

नोटपैड को निम्न प्राथमिकता के साथ प्रारंभ करता है और तुरंत वापस आता है।

### Example 2

```powershell
nice notepad.exe -Priority High
```

उच्च प्राथमिकता के साथ एक प्रक्रिया शुरू करने के लिए 'nice' उपनाम का उपयोग करता है।
