# Start-ProcessWithPriority

> **Module:** GenXdev.Windows | **Type:** Cmdlet | **Aliases:** `nice`

## Synopsis

> Memulai proses dengan tingkat prioritas yang ditentukan.

## Description

* Meluncurkan eksekusi dengan tingkat prioritas yang dapat disesuaikan dan menyediakan opsi untuk menunggu dan penanganan proses.
* Membungkus Start-Process dengan fungsionalitas tambahan untuk mengontrol prioritas proses dan perilaku eksekusi.

## Syntax

```powershell
Start-ProcessWithPriority -FilePath <String> [[-ArgumentList] <String[]>] [[-Priority] <String>] [-NoWait] [-PassThru] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-FilePath` | String | ✅ | — | 0 | — | Jalur ke file yang dapat dijalankan |
| `-ArgumentList` | String[] | — | — | 1 | — | Argumen yang akan diteruskan ke file yang dapat dieksekusi |
| `-Priority` | String | — | — | 2 | — | Tingkat prioritas proses |
| `-NoWait` | SwitchParameter | — | — | Named | `False` | Jangan menunggu penyelesaian proses |
| `-PassThru` | SwitchParameter | — | — | Named | `False` | Kembalikan objek proses |

## Examples

### Example 1

```powershell
Start-ProcessWithPriority -FilePath "notepad.exe" -Priority "Low" -NoWait
```

Memulai Notepad dengan prioritas rendah dan segera kembali.

### Example 2

```powershell
nice notepad.exe -Priority High
```

Menggunakan alias 'nice' untuk memulai proses dengan prioritas tinggi.
