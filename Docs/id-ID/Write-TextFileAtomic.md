# Write-TextFileAtomic

> **Module:** GenXdev.FileSystem | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> Menulis teks ke file secara atomik untuk mencegah kerusakan.

## Description

* Penulisan atomik: menggunakan strategi file-temp + rename untuk memastikan file target tidak pernah dalam keadaan rusak jika proses terputus.
* Logika pengulangan: mengulangi penulisan hingga MaxRetries kali dengan jeda RetryDelayMs milidetik antar percobaan.
* Dukungan encoding: menerima parameter System.Text.Encoding untuk mengontrol encoding keluaran. Standarnya UTF-8.
* Dukungan debounce: ketika DebounceMs > 0, penulisan berturut-turut ke file yang sama akan digabungkan — hanya payload terakhir yang ditulis setelah file tidak disentuh selama DebounceMs ms.
* Pembuatan direktori: membuat direktori induk secara otomatis jika belum ada.

## Syntax

```powershell
Write-TextFileAtomic -FilePath <String> [[-Data] <String>] [-DebounceMs <Int32>] [-Encoding <Text.Encoding>] [-MaxRetries <Int32>] [-RetryDelayMs <Int32>] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-FilePath` | String | ✅ | — | 0 | — | Jalur ke file yang akan ditulis. |
| `-Data` | String | — | — | 1 | — | Teks konten yang akan ditulis ke file. |
| `-MaxRetries` | Int32 | — | — | Named | `0` | Jumlah maksimum percobaan ulang untuk operasi penulisan atomik. Default adalah 4. |
| `-RetryDelayMs` | Int32 | — | — | Named | `0` | Penundaan dalam milidetik antara percobaan ulang. Default adalah 200. |
| `-Encoding` | Text.Encoding | — | — | Named | — | Encoding teks yang akan digunakan saat menulis file. Default adalah UTF-8. |
| `-DebounceMs` | Int32 | — | — | Named | `0` | Jika lebih besar dari 0, debounce menulis ke file ini. Default adalah 0 (dinonaktifkan). |

## Examples

### Example 1

```powershell
Write-TextFileAtomic -FilePath "config.txt" -Data "hello world"
```

Menulis string ke config.txt menggunakan pengkodean UTF-8 secara atomik.

### Example 2

```powershell
# Debounce: only the LAST log line within 2 seconds hits disk
Get-Content server.log -Tail 50 | ForEach-Object {
    Write-TextFileAtomic -FilePath "filtered.log" -Data $_ `
        -DebounceMs 2000
}
Start-Sleep -Seconds 3
```

Mengalirkan 50 baris dengan cepat; hanya baris terakhir yang dipertahankan setelah periode hening 2 detik.

### Example 3

```powershell
Write-TextFileAtomic -FilePath "data.csv" -Data $csv `
    -Encoding ([System.Text.Encoding]::ASCII)
```

Menulis string ke data.csv menggunakan pengkodean ASCII secara atomik.

### Example 4

```powershell
# Debounce with custom retry: coalesce UI state saves
$watcher = Start-ThreadJob -ScriptBlock {
    param($path)
    while ($true) {
        Write-TextFileAtomic -FilePath $path `
            -Data (Get-Date -Format 'yyyy-MM-dd HH:mm:ss') `
            -DebounceMs 1000 -MaxRetries 3
        Start-Sleep -Milliseconds 100
    }
} -ArgumentList "timestamp.txt"
```

Sebuah pekerjaan latar menulis cap waktu setiap 100ms tetapi debounce menggabungkannya — hanya satu tulis per detik yang benar-benar mencapai disk.
