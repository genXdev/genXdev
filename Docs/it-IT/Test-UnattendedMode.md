# Test-UnattendedMode

> **Module:** GenXdev.Helpers | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> Rileva se PowerShell è in esecuzione in modalità non presenziata/automatica.

## Description

* Analizza vari indicatori per determinare se PowerShell è in esecuzione in un contesto non presidiato o automatizzato.
* Controlla l'analisi della pipeline, le variabili d'ambiente, il reindirizzamento della console e il contesto di chiamata.
* Quando viene fornito CallersInvocation, analizza la posizione e il conteggio nella pipeline per rilevare l'esecuzione automatizzata di pipeline o script.

## Syntax

```powershell
Test-UnattendedMode [[-CallersInvocation] <InvocationInfo>] [-Detailed] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-CallersInvocation` | InvocationInfo | — | — | 0 | — | Informazioni di chiamata del chiamante per il rilevamento di pipeline e automazione |
| `-Detailed` | SwitchParameter | — | — | Named | `False` | Restituisci un oggetto di analisi dettagliato invece di un semplice booleano |

## Examples

### Example 1

```powershell
Test-UnattendedMode
```

Restituisce un valore booleano che indica se è in esecuzione in modalità non presidiata.

### Example 2

```powershell
Test-UnattendedMode -CallersInvocation $MyInvocation
```

Analizza il contesto di chiamata del chiamante e restituisce un valore booleano.

### Example 3

```powershell
Test-UnattendedMode -CallersInvocation $MyInvocation -Detailed
```

Restituisce un oggetto di analisi dettagliato con tutti gli indicatori.

### Example 4

```powershell
function My-Function {
    $isUnattended = Test-UnattendedMode -CallersInvocation $MyInvocation
    if ($isUnattended) {
        Write-Verbose "Running in unattended mode"
    }
}
```

Esempio di utilizzo in una funzione per verificare la modalità non presidiata.
