# Add-FeatureLineToREADME

> **Module:** GenXdev.Coding | **Type:** Function | **Aliases:** `feature`

## Synopsis

> Voegt een functieregel toe aan het README-bestand met een tijdstempel.

## Description

Voegt een functieregel toe aan het opgegeven README-bestand, voorafgegaan door de huidige datum in yyyyMMdd-indeling. De regel kan worden opgemaakt als code en optioneel worden weergegeven.

## Syntax

```powershell
Add-FeatureLineToREADME [[-Line] <String>] [-Ascending] [-Code] [-First <Int32>] [-Priority <Int32>] [-Show] [-SortByDate] [-UseHomeREADME] [-UseOneDriveREADME] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-Line` | String | — | — | 0 | `''` | De functiebeschrijvingstekst om toe te voegen |
| `-Code` | SwitchParameter | — | — | Named | — | Formatteer de regel als code |
| `-Show` | SwitchParameter | — | — | Named | — | Geef de README weer na het toevoegen van de regel |
| `-UseHomeREADME` | SwitchParameter | — | — | Named | — | Gebruik README in de thuismap |
| `-UseOneDriveREADME` | SwitchParameter | — | — | Named | — | Gebruik README in OneDrive-map |
| `-Priority` | Int32 | — | — | Named | `1` | Prioriteit voor sorteren (hoger = eerst weergeven, standaard 1) |
| `-SortByDate` | SwitchParameter | — | — | Named | — | Regel sorteren op datum (yyyymmdd-voorvoegsel) in plaats van prioriteit |
| `-Ascending` | SwitchParameter | — | — | Named | — | De sorteervolgorde omkeren (oplopend in plaats van aflopend) |
| `-First` | Int32 | — | — | Named | `0` | Limiet - Toon alleen de eerste N regels |

## Examples

### Add-FeatureLineToREADME -Line "Added new Git feature"

```powershell
Add-FeatureLineToREADME -Line "Added new Git feature"
```

### feature "Added new Git feature" -Code -Show

```powershell
feature "Added new Git feature" -Code -Show
```
