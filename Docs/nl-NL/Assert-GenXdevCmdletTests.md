# Assert-GenXdevCmdletTests

> **Module:** GenXdev.Coding.PowerShell.Modules | **Type:** Function | **Aliases:** `improvecmdlettests`

## Synopsis

> Controleert en verbetert eenheidstests van een opgegeven GenXdev-cmdlet.

## Description

Deze functie helpt bij het onderhouden en verbeteren van unittesten voor GenXdev-cmdlets door:
1. Testbestanden aan te maken als ze nog niet bestaan
2. De cmdlet te openen in VS Code
3. AI-prompts voor te bereiden en toe te passen voor het genereren/verbeteren van testen
4. De uitvoeringsworkflow van testen te beheren

## Syntax

```powershell
Assert-GenXdevCmdletTests -CmdletName <String> [[-Prompt] <String>] [[-PromptKey] <String>] [-AssertFailedTest] [-ContinuationHandled] [-EditPrompt] [-FromScripts] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-CmdletName` | String | ✅ | ✅ (ByValue, ByPropertyName) | 0 | — | De naam van de cmdlet om unit-tests voor te verbeteren |
| `-Prompt` | String | — | — | 1 | `''` | Aangepaste AI-prompttekst om te gebruiken |
| `-PromptKey` | String | — | — | 2 | — | De AI-prompt-sleutel die moet worden gebruikt voor sjabloonselectie |
| `-EditPrompt` | SwitchParameter | — | — | Named | — | Schakel over naar alleen de AI-prompt bewerken |
| `-AssertFailedTest` | SwitchParameter | — | — | Named | — | Geeft aan dat een mislukte test is bevestigd |
| `-FromScripts` | SwitchParameter | — | — | Named | — | Zoeken in scriptbestanden in plaats van in modules |
| `-ContinuationHandled` | SwitchParameter | — | — | Named | — | The `-ContinuationHandled` parameter. |
