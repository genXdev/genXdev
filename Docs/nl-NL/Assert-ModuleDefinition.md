# Assert-ModuleDefinition

> **Module:** GenXdev.Coding.PowerShell.Modules | **Type:** Function | **Aliases:** —

## Synopsis

> Helpt bij het refactoren van PowerShell broncodebestanden met behulp van AI-ondersteuning.

## Description

Deze functie automatiseert het proces van het herstructureren van PowerShell-code met behulp van AI.
Het beheert promptsjablonen, detecteert de actieve IDE (VS Code of Visual Studio),
en orkestreert de herstructureringsworkflow via toetsenbordautomatisering.
De functie kan zowel modulemanifest- (.psd1) als modulescriptbestanden (.psm1) verwerken.

## Syntax

```powershell
Assert-ModuleDefinition -ModuleName <String> [-EditPrompt] [-Prompt <String>] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-ModuleName` | String | ✅ | ✅ (ByValue, ByPropertyName) | 0 | — | De naam van de module |
| `-Prompt` | String | — | — | Named | — | Als u een computer gebruikt, moet u eerst het volgende doen: 
- Open PowerShell als administrator
- Voer de opdracht `Set-ExecutionPolicy RemoteSigned` uit
- Bevestig met `Y` of `A`

Daarna kunt u:
```powershell
Install-Module -Name PSWindowsUpdate -Force
```
uitvoeren om de module te installeren. |
| `-EditPrompt` | SwitchParameter | — | — | Named | — | Schakel over naar alleen de AI-prompt bewerken |
