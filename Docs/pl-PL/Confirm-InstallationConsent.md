# Confirm-InstallationConsent

> **Module:** GenXdev.Helpers | **Type:** Function | **Aliases:** —

## Synopsis

> Potwierdza zgodę użytkownika na instalację oprogramowania firm trzecich, wykorzystując preferencje do trwałych wyborów.

## Description

<details>
<summary><b>Expand description</b></summary>

Ta funkcja działa jako niestandardowy mechanizm ShouldProcess przeznaczony specjalnie do instalacji oprogramowania. Sprawdza preferencje użytkownika, aby określić, czy automatyczna instalacja jest dozwolona dla danej aplikacji. Jeśli preferencja nie jest ustawiona, wyświetla użytkownikowi jasne wyjaśnienie, co zostanie zainstalowane, źródło, potencjalne ryzyko oraz opcje zezwolenia lub odrzucenia instalacji (z możliwością wyboru jednorazowego lub stałego).

Zapewnia to wyraźną zgodę użytkownika przed przystąpieniem do jakiejkolwiek instalacji, pomagając zminimalizować potencjalne ryzyko prawne poprzez wymaganie potwierdzenia działania przez użytkownika. Komunikat jasno stwierdza, że autor modułu (GenXdev) nie ponosi odpowiedzialności za oprogramowanie firm trzecich, a użytkownik wyraża zgodę na instalację na własne ryzyko.

Preferencje są przechowywane w pliku JSON, umożliwiając użytkownikom ustawienie „zawsze zezwalaj” dla konkretnych aplikacji lub włączenie globalnej zgody na wszystkie instalacje GenXdev firm trzecich, co jest wygodne, a jednocześnie zgodne z prawem.

Jeśli zgoda zostanie odrzucona (lub preferencja jest ustawiona na odmowę), funkcja zwraca $false i nie przystępuje do instalacji. Jeśli dozwolona, zwraca $true.

</details>
## Syntax

```powershell
Confirm-InstallationConsent -ApplicationName <String> -Source <String> [-ConsentToThirdPartySoftwareInstallation] [-Description <String>] [-ForceConsent] [-Publisher <String>] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-ApplicationName` | String | ✅ | — | 0 | — | Nazwa instalowanej aplikacji lub oprogramowania. |
| `-Source` | String | ✅ | — | 1 | — | Źródło instalacji (np. Winget, PowerShell Gallery). |
| `-Description` | String | — | — | Named | `"This software is required for certain features in the GenXdev modules."` | Opcjonalny opis oprogramowania i jego przeznaczenia. |
| `-Publisher` | String | — | — | Named | `"Third-party vendor"` | Opcjonalny wydawca lub dostawca oprogramowania. |
| `-ForceConsent` | SwitchParameter | — | — | Named | — | Wymuś monit, nawet jeśli preferencja jest ustawiona. |
| `-ConsentToThirdPartySoftwareInstallation` | SwitchParameter | — | — | Named | — | Automatycznie wyrażaj zgodę na instalację oprogramowania firm trzecich i ustaw trwałą flagę. |

## Examples

### if (Confirm-InstallationConsent -ApplicationName "Docker Desktop" `         -Source "Winget") {     # Proceed with installation     Microsoft.WinGet.Client\Install-WinGetPackage `         -Id "Docker.DockerDesktop" }

```powershell
if (Confirm-InstallationConsent -ApplicationName "Docker Desktop" `
        -Source "Winget") {
    # Proceed with installation
    Microsoft.WinGet.Client\Install-WinGetPackage `
        -Id "Docker.DockerDesktop"
}
```

Sprawdza to zgodę przed zainstalowaniem Docker Desktop za pomocą Winget.

### Confirm-InstallationConsent -ApplicationName "Pester" `     -Source "PowerShell Gallery" -Publisher "Pester Team" `     -Description "Required for unit testing in PowerShell modules."

```powershell
Confirm-InstallationConsent -ApplicationName "Pester" `
    -Source "PowerShell Gallery" -Publisher "Pester Team" `
    -Description "Required for unit testing in PowerShell modules."
```

Monity z szczegółowymi informacjami przed instalacją modułu Pester.
