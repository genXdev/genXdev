# Get-RefactorReport

> **Module:** GenXdev.Coding | **Type:** Cmdlet | **Aliases:** `refactorreport`

## Synopsis

> Generuje szczegółowy raport operacji refaktoryzacji i ich stanu.

## Description

Analizuje i raportuje postęp operacji refaktoryzacji, badając ich bieżący stan, status ukończenia oraz dotknięte funkcje. Dostarcza wyniki w ustrukturyzowanym formacie hashtable lub czytelnych dla człowieka kolumnach tekstowych. Raport zawiera nazwę refaktora, klucz monitu, priorytet, status, liczbę funkcji i procent ukończenia.

## Syntax

```powershell
Get-RefactorReport [[-Name] <String[]>] [-AsText] [-ClearSession] [-PreferencesDatabasePath <String>] [-SessionOnly] [-SkipSession] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-Name` | String[] | — | ✅ (ByValue, ByPropertyName) | 0 | — | Nazwa refaktoryzacji akceptuje symbole wieloznaczne 🌐 *Supports wildcards* |
| `-PreferencesDatabasePath` | String | — | — | Named | — | Określa ścieżkę do pliku bazy danych preferencji. |
| `-SessionOnly` | SwitchParameter | — | — | Named | `False` | Jeśli ustawiono, używaj tylko pamięci podręcznej sesji dla danych refaktora. |
| `-ClearSession` | SwitchParameter | — | — | Named | `False` | Jeśli ustawiono, wyczyść pamięć podręczną sesji przed uruchomieniem. |
| `-SkipSession` | SwitchParameter | — | — | Named | `False` | Jeśli ustawiony, pomiń ładowanie pamięci podręcznej sesji. |
| `-AsText` | SwitchParameter | — | — | Named | `False` | Wyprowadź raport w formacie tekstowym zamiast jako tablicę mieszającą. |

## Examples

### Example 1

```powershell
Get-RefactorReport -Name "DatabaseRefactor" -AsText
```

Generuje raport tekstowy dla refaktoryzacji pasujących do „DatabaseRefactor”.

### Example 2

```powershell
refactorreport "*"
```

Generuje raport hashtable dla wszystkich refaktoryzacji używających aliasu.
