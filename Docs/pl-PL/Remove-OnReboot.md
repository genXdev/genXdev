# Remove-OnReboot

> **Module:** GenXdev.FileSystem | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> Oznacza pliki lub katalogi do usunięcia podczas następnego uruchomienia systemu.

## Description

* Używa interfejsu API systemu Windows do oznaczania plików do usunięcia przy następnym uruchomieniu komputera.
* Obsługuje zablokowane pliki, najpierw próbując zmienić ich nazwy na tymczasowe, a następnie śledząc wszystkie przeniesienia w celu zachowania integralności systemu plików.
* Jeśli zmiana nazwy się nie powiedzie, można użyć parametru -MarkInPlace, aby oznaczyć pliki w ich oryginalnej lokalizacji.

## Syntax

```powershell
Remove-OnReboot -Path <String[]> [-MarkInPlace] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-Path` | String[] | ✅ | ✅ (ByValue) | 0 | — | Ścieżki do plików/katalogów do oznaczenia do usunięcia |
| `-MarkInPlace` | SwitchParameter | — | — | Named | `False` | Oznacza pliki do usunięcia bez zmiany nazwy |

## Examples

### Example 1

```powershell
Remove-OnReboot -Path "C:\temp\locked-file.txt"
```

Oznacza zablokowany plik do usunięcia podczas następnego uruchomienia systemu.

### Example 2

```powershell
"file1.txt","file2.txt" | Remove-OnReboot -MarkInPlace
```

Oznacza wiele plików do usunięcia, używając MarkInPlace dla tych, których nie można zmienić nazwy.
