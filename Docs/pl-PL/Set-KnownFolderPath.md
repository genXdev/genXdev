# Set-KnownFolderPath

> **Module:** GenXdev.Windows | **Type:** Function | **Aliases:** —

## Synopsis

> Modyfikuje fizyczną ścieżkę znanego folderu systemu Windows.

## Description

Używa interfejsu API Shell32 systemu Windows do przenoszenia folderów systemowych, takich jak Dokumenty, Pobrane, Pulpit lub innych znanych folderów systemu Windows, do nowej lokalizacji. Funkcja sprawdza, czy ścieżka docelowa istnieje, wyszukuje unikatowy identyfikator GUID folderu w rejestrze znanych folderów i używa interfejsu API SHSetKnownFolderPath do wykonania przeniesienia. Typowe przypadki użycia obejmują przenoszenie folderów użytkownika na inny dysk w celu zarządzania miejscem lub organizacji. Należy zachować ostrożność podczas przenoszenia folderów krytycznych dla systemu, ponieważ może to wpłynąć na stabilność systemu.

## Syntax

```powershell
Set-KnownFolderPath -KnownFolder <String> -Path <String> [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-KnownFolder` | String | ✅ | — | 0 | — | Znany folder do modyfikacji |
| `-Path` | String | ✅ | — | 1 | — | Nowa ścieżka dla znanego folderu |

## Examples

### Set-KnownFolderPath -KnownFolder 'Documents' -Path 'D:\UserDocs'

```powershell
Set-KnownFolderPath -KnownFolder 'Documents' -Path 'D:\UserDocs'
```

### Set-KnownFolderPath Downloads 'E:\Downloads'

```powershell
Set-KnownFolderPath Downloads 'E:\Downloads'
```
