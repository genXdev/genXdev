# Get-PowershellMainWindowProcess

> **Module:** GenXdev.Windows | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> Возвращает объект процесса для окна, в котором размещен терминал PowerShell.

## Description

* Обходит дерево процессов вверх, начиная с текущего процесса PowerShell,
  чтобы найти родительское окно, ответственное за размещение терминала.
* Если непосредственный родительский процесс не имеет дескриптора главного окна,
  ищет аналогичные процессы, которые имеют главные окна.
* Полезно для определения фактического процесса окна терминала (например, Windows
  Terminal, ConHost и т.д.), который содержит сеанс PowerShell.

## Syntax

```powershell
Get-PowershellMainWindowProcess [<CommonParameters>]
```

## Examples

### Example 1

```powershell
$hostProcess = Get-PowershellMainWindowProcess
Write-Host "PowerShell is hosted in: $($hostProcess.ProcessName)"
```

Возвращает процесс, в котором размещается текущий сеанс PowerShell, и отображает его имя.
