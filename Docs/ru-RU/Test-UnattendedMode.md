# Test-UnattendedMode

> **Module:** GenXdev.Helpers | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> Определяет, выполняется ли PowerShell в автоматическом/неинтерактивном режиме.

## Description

* Анализирует различные индикаторы, чтобы определить, выполняется ли PowerShell в
  автоматическом или автоматизированном контексте.
* Проверяет анализ конвейера, переменные среды, перенаправление консоли
  и контекст вызова.
* Если предоставлен CallersInvocation, анализирует позицию в конвейере и
  количество для обнаружения автоматизированного выполнения конвейера или сценария.

## Syntax

```powershell
Test-UnattendedMode [[-CallersInvocation] <InvocationInfo>] [-Detailed] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-CallersInvocation` | InvocationInfo | — | — | 0 | — | Информация о вызове вызывающего объекта для обнаружения конвейера и автоматизации |
| `-Detailed` | SwitchParameter | — | — | Named | `False` | Возвращайте детальный объект анализа вместо простого логического значения |

## Examples

### Example 1

```powershell
Test-UnattendedMode
```

Возвращает логическое значение, указывающее, выполняется ли работа в автоматическом режиме.

### Example 2

```powershell
Test-UnattendedMode -CallersInvocation $MyInvocation
```

Анализирует контекст вызова вызывающей стороны и возвращает логическое значение.

### Example 3

```powershell
Test-UnattendedMode -CallersInvocation $MyInvocation -Detailed
```

Возвращает подробный объект анализа со всеми индикаторами.

### Example 4

```powershell
function My-Function {
    $isUnattended = Test-UnattendedMode -CallersInvocation $MyInvocation
    if ($isUnattended) {
        Write-Verbose "Running in unattended mode"
    }
}
```

Пример использования в функции для проверки автоматического режима.
