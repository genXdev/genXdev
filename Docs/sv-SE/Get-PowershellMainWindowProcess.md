# Get-PowershellMainWindowProcess

> **Module:** GenXdev.Windows | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> Returnerar processobjektet för fönstret som är värd för PowerShell-terminalen.

## Description

* Går uppåt i processträdet från den aktuella PowerShell-processen
  för att hitta det överordnade fönster som är ansvarigt för att vara värd för terminalen.
* Om den omedelbara överordnade processen inte har ett huvudfönsterhandtag
  söker den efter liknande processer som har huvudfönster.
* Användbart för att identifiera den faktiska terminalfönsterprocessen (som Windows
  Terminal, ConHost, etc.) som innehåller PowerShell-sessionen.

## Syntax

```powershell
Get-PowershellMainWindowProcess [<CommonParameters>]
```

## Examples

### Example 1

```powershell
$hostProcess = Get-PowershellMainWindowProcess
Write-Host "PowerShell is hosted in: $($hostProcess.ProcessName)"
```

Returnerar processen som är värd för den aktuella PowerShell-sessionen och visar dess namn.
