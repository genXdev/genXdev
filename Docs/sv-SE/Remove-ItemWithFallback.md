# Remove-ItemWithFallback

> **Module:** GenXdev.FileSystem | **Type:** Function | **Aliases:** `rmf`

## Synopsis

> Tar bort filer eller kataloger med flera reservmekanismer för pålitlig borttagning.

## Description

Den här funktionen ger ett robust sätt att ta bort filer och kataloger genom att försöka med flera borttagningsmetoder i följd:
1. Direkt borttagning via System.IO-metoder för bästa prestanda
2. PowerShell-provider-medveten Remove-Item-cmdlet som reserv
3. Markera för borttagning vid nästa systemomstart om andra metoder misslyckas
Detta säkerställer maximal tillförlitlighet vid borttagning av objekt över olika providers.

## Syntax

```powershell
Remove-ItemWithFallback -Path <String> [-CountRebootDeletionAsSuccess] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-Path` | String | ✅ | ✅ (ByValue, ByPropertyName) | 0 | — | Sökvägen till objektet som ska tas bort |
| `-CountRebootDeletionAsSuccess` | SwitchParameter | — | — | Named | `$false` | The `-CountRebootDeletionAsSuccess` parameter. |

## Examples

### Remove-ItemWithFallback -Path "C:\temp\myfile.txt" Attempts to remove the file using all available methods.

```powershell
Remove-ItemWithFallback -Path "C:\temp\myfile.txt"
Attempts to remove the file using all available methods.
```

### "C:\temp\mydir" | rif Uses the alias 'rif' to remove a directory through the pipeline.

```powershell
"C:\temp\mydir" | rif
Uses the alias 'rif' to remove a directory through the pipeline.
```
