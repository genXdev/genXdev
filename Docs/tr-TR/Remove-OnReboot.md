# Remove-OnReboot

> **Module:** GenXdev.FileSystem | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> Bir sonraki sistem başlangıcında silinmek üzere dosya veya dizinleri işaretler.

## Description

* Dosyaları bir sonraki açılışta silinmek üzere işaretlemek için Windows API'sini kullanır.
* Kilitli dosyaları önce geçici adlarla yeniden adlandırmaya çalışarak işler ve dosya sistemi bütünlüğünü korumak için tüm taşımaları izler.
* Yeniden adlandırma başarısız olursa, dosyaları orijinal konumlarında işaretlemek için -MarkInPlace parametresi kullanılabilir.

## Syntax

```powershell
Remove-OnReboot -Path <String[]> [-MarkInPlace] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-Path` | String[] | ✅ | ✅ (ByValue) | 0 | — | Silinmek üzere işaretlenecek dosya/dizin yolları |
| `-MarkInPlace` | SwitchParameter | — | — | Named | `False` | Yeniden adlandırmadan silme için dosyaları işaretler |

## Examples

### Example 1

```powershell
Remove-OnReboot -Path "C:\temp\locked-file.txt"
```

Sistem bir sonraki başlangıcında kilitli bir dosyayı silinmek üzere işaretler.

### Example 2

```powershell
"file1.txt","file2.txt" | Remove-OnReboot -MarkInPlace
```

Yeniden adlandırılamayan dosyalar için MarkInPlace kullanarak birden çok dosyayı silinmek üzere işaretler.
