# Assert-ModuleDefinition

> **Module:** GenXdev.Coding.PowerShell.Modules | **Type:** Function | **Aliases:** —

## Synopsis

> Допомагає у рефакторингу файлів вихідного коду PowerShell за допомогою штучного інтелекту.

## Description

Ця функція автоматизує процес рефакторингу коду PowerShell за допомогою штучного інтелекту.
Вона керує шаблонами запитів, визначає активне IDE (VS Code або Visual Studio)
та оркеструє робочий процес рефакторингу через автоматизацію клавіатури.
Функція може працювати як з файлами маніфесту модуля (.psd1), так і з файлами сценарію модуля (.psm1).

## Syntax

```powershell
Assert-ModuleDefinition -ModuleName <String> [-EditPrompt] [-Prompt <String>] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-ModuleName` | String | ✅ | ✅ (ByValue, ByPropertyName) | 0 | — | Назва модуля |
| `-Prompt` | String | — | — | Named | — | Під-ключі реєстру для параметрів команди не знайдено. |
| `-EditPrompt` | SwitchParameter | — | — | Named | — | Переключіться на редагування лише підказки ШІ |

## Examples

### Assert-ModuleDefinition -ModuleName "MyModule" -EditPrompt

```powershell
Assert-ModuleDefinition -ModuleName "MyModule" -EditPrompt
```

### "MyModule" | Assert-ModuleDefinition

```powershell
"MyModule" | Assert-ModuleDefinition
```
