# Add-TodoLineToREADME

> **Module:** GenXdev.Coding | **Type:** Function | **Aliases:** `todo`

## Synopsis

> 在 README.md 文件中添加一个待办事项。

## Description

向 README.md 文件的 "## Todoos" 部分添加一个带时间戳的待办事项。
待办事项可以标记为已完成，并显示修改后的部分。
除非标记为已完成，否则每个新待办事项都会自动添加时间戳。

## Syntax

```powershell
Add-TodoLineToREADME [[-Line] <String>] [-Ascending] [-Code] [-Done] [-First <Int32>] [-Priority <Int32>] [-Show] [-SortByDate] [-UseHomeREADME] [-UseOneDriveREADME] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-Line` | String | — | — | 0 | `''` | 要添加的待办事项文本 |
| `-Code` | SwitchParameter | — | — | Named | — | 在 Visual Studio Code 中打开 README |
| `-Show` | SwitchParameter | — | — | Named | — | 显示已修改的部分 |
| `-Done` | SwitchParameter | — | — | Named | — | 将待办事项标记为已完成 |
| `-UseHomeREADME` | SwitchParameter | — | — | Named | — | 在 PowerShell 配置文件中使用 README |
| `-UseOneDriveREADME` | SwitchParameter | — | — | Named | — | 在 OneDrive 目录中使用 README |
| `-Priority` | Int32 | — | — | Named | `1` | 排序优先级（值越高越靠前，默认为1） |
| `-SortByDate` | SwitchParameter | — | — | Named | — | 按日期（yyyyMMdd前缀）排序行，而不是按优先级 |
| `-Ascending` | SwitchParameter | — | — | Named | — | 颠倒排序顺序（升序而非降序） |
| `-First` | Int32 | — | — | Named | `0` | 限制 - 仅显示前 N 行输出 |

## Examples

### Add-TodoLineToREADME -Line "Implement new feature" -Show -Code

```powershell
Add-TodoLineToREADME -Line "Implement new feature" -Show -Code
```

### todo "Fix bug" -Done

```powershell
todo "Fix bug" -Done
```
