# Assert-GenXdevCmdletTests

> **Module:** GenXdev.Coding.PowerShell.Modules | **Type:** Function | **Aliases:** `improvecmdlettests`

## Synopsis

> 断言并改进指定的GenXdev cmdlet的单元测试。

## Description

此函数通过以下方式帮助维护和改进 GenXdev cmdlet 的单元测试：
1. 如果测试文件不存在，则创建它们
2. 在 VS Code 中打开 cmdlet
3. 准备并应用 AI 提示以生成/改进测试
4. 管理测试执行工作流

## Syntax

```powershell
Assert-GenXdevCmdletTests -CmdletName <String> [[-Prompt] <String>] [[-PromptKey] <String>] [-AssertFailedTest] [-ContinuationHandled] [-EditPrompt] [-FromScripts] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-CmdletName` | String | ✅ | ✅ (ByValue, ByPropertyName) | 0 | — | 用于改进单元测试的 cmdlet 名称 |
| `-Prompt` | String | — | — | 1 | `''` | 自定义 AI 提示文本 |
| `-PromptKey` | String | — | — | 2 | — | 用于模板选择的AI提示键 |
| `-EditPrompt` | SwitchParameter | — | — | Named | — | 仅编辑AI提示词 |
| `-AssertFailedTest` | SwitchParameter | — | — | Named | — | 表示断言测试失败 |
| `-FromScripts` | SwitchParameter | — | — | Named | — | 在脚本文件中搜索，而非模块 |
| `-ContinuationHandled` | SwitchParameter | — | — | Named | — | The `-ContinuationHandled` parameter. |
