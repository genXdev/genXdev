# Confirm-InstallationConsent

> **Module:** GenXdev.Helpers | **Type:** Function | **Aliases:** —

## Synopsis

> 确认用户同意安装第三方软件时，使用偏好设置以进行持久化选择。

## Description

<details>
<summary><b>Expand description</b></summary>

此函数作为自定义的ShouldProcess机制，专门用于软件安装。它检查用户偏好设置，以确定是否允许自动安装指定应用程序。若未设置偏好，则会向用户显示清晰说明，包括将要安装的内容、来源、潜在风险，以及允许或拒绝安装的选项（支持一次性或持久选择）。

这确保了在任何安装进行前获得用户的明确同意，通过要求用户主动操作，有助于降低潜在的法律风险。提示中明确声明模块作者（GenXdev）不对第三方软件负责，且用户需自行承担安装风险。

偏好设置存储于JSON文件中，用户可为特定应用设置“始终允许”，或启用对所有GenXdev第三方安装的全局同意，既方便操作又符合法律要求。

若用户拒绝同意（或偏好设置为拒绝），则函数返回$false，不继续安装；若允许，则返回$true。

</details>
## Syntax

```powershell
Confirm-InstallationConsent -ApplicationName <String> -Source <String> [-ConsentToThirdPartySoftwareInstallation] [-Description <String>] [-ForceConsent] [-Publisher <String>] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-ApplicationName` | String | ✅ | — | 0 | — | 正在安装的应用程序或软件的名称。 |
| `-Source` | String | ✅ | — | 1 | — | 安装源（例如，Winget、PowerShell Gallery）。 |
| `-Description` | String | — | — | Named | `"This software is required for certain features in the GenXdev modules."` | 软件及其用途的可选描述。 |
| `-Publisher` | String | — | — | Named | `"Third-party vendor"` | 软件的发布者或供应商（可选）。 |
| `-ForceConsent` | SwitchParameter | — | — | Named | — | 即使设置了首选项，也强制显示提示。 |
| `-ConsentToThirdPartySoftwareInstallation` | SwitchParameter | — | — | Named | — | 自动同意第三方软件安装并设置持久化标志。 |

## Examples

### if (Confirm-InstallationConsent -ApplicationName "Docker Desktop" `         -Source "Winget") {     # Proceed with installation     Microsoft.WinGet.Client\Install-WinGetPackage `         -Id "Docker.DockerDesktop" }

```powershell
if (Confirm-InstallationConsent -ApplicationName "Docker Desktop" `
        -Source "Winget") {
    # Proceed with installation
    Microsoft.WinGet.Client\Install-WinGetPackage `
        -Id "Docker.DockerDesktop"
}
```

这会在通过 Winget 安装 Docker Desktop 之前检查同意情况。

### Confirm-InstallationConsent -ApplicationName "Pester" `     -Source "PowerShell Gallery" -Publisher "Pester Team" `     -Description "Required for unit testing in PowerShell modules."

```powershell
Confirm-InstallationConsent -ApplicationName "Pester" `
    -Source "PowerShell Gallery" -Publisher "Pester Team" `
    -Description "Required for unit testing in PowerShell modules."
```

安装 Pester 模块前提供详细信息的提示。
