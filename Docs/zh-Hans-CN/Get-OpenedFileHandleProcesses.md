# Get-OpenedFileHandleProcesses

> **Module:** GenXdev.Windows | **Type:** Cmdlet | **Aliases:** —

## Synopsis

> 检索对指定文件持有打开句柄的进程。

## Description

* 使用 Sysinternals 的 handle.exe 工具来识别当前对一个或多个指定文件具有打开句柄的进程。
* 有助于确定哪些进程阻止了删除或修改等文件操作。
* 需要安装 Sysinternals 套件中的 handle.exe，并确保其在系统路径中可用。

## Syntax

```powershell
Get-OpenedFileHandleProcesses -FilePath <String[]> [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-FilePath` | String[] | ✅ | ✅ (ByValue, ByPropertyName) | 0 | — | 要检查是否有打开句柄的文件的路径 |

## Examples

### Example 1

```powershell
Get-OpenedFileHandleProcesses -FilePath "C:\temp\example.txt"
```

标识所有对指定文件持有打开句柄的进程。

### Example 2

```powershell
"file1.txt", "file2.txt" | Get-OpenedFileHandleProcesses
```

通过管道输入检查多个文件，查找具有打开句柄的进程。
