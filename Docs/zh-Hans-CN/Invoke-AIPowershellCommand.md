# Invoke-AIPowershellCommand

> **Module:** GenXdev.AI.Queries | **Type:** Function | **Aliases:** `hint`

## Synopsis

> 使用 AI 辅助生成并执行 PowerShell 命令。

## Description

使用AI模型根据自然语言查询生成PowerShell命令。该函数可以直接将命令发送到PowerShell窗口，也可以将其复制到剪贴板。它利用AI模型解释自然语言并将其转换为可执行的PowerShell命令，并为各种AI后台提供全面的参数支持。

## Syntax

```powershell
Invoke-AIPowershellCommand -PowershellCmdline <String> [-ApiEndpoint <String>] [-ApiKey <String>] [-LLMQueryType <String>] [-Model <String>] [-NoSupportForJsonSchema <String>] [<CommonParameters>]
```

## Parameters

| Name | Type | Required | Pipeline | Position | Default | Description |
|:---|:---|:---:|:---|:---:|:---|:---|
| `-PowershellCmdline` | String | ✅ | — | 0 | — | 用于生成命令的自然语言查询 |
| `-LLMQueryType` | String | — | — | Named | `'Coding'` | LLM 查询的类型 |
| `-Model` | String | — | — | Named | — | 用于AI操作的模型标识符或模式 |
| `-ApiEndpoint` | String | — | — | Named | — | 用于AI操作的API端点URL |
| `-ApiKey` | String | — | — | Named | — | 用于认证AI操作的API密钥 |
| `-NoSupportForJsonSchema` | String | — | — | Named | — | 表示 LLM 不支持 JSON 模式 |

## Examples

### Invoke-AIPowershellCommand -Query "list all running processes"

```powershell
Invoke-AIPowershellCommand -Query "list all running processes"
```

生成列出正在运行的进程的 PowerShell 命令

### hint "list files modified today"

```powershell
hint "list files modified today"
```

使用别名生成查找今天修改的文件的命令。

### Invoke-AIPowershellCommand -Query "stop service" -Clipboard

```powershell
Invoke-AIPowershellCommand -Query "stop service" -Clipboard
```

生成一条停止服务的命令并将其复制到剪贴板。
